- **Class Extension:** Add `WorldBounds` (minX, maxX, minY, maxY) to `ApplicationView`.

- **DeviceToWorldXform (Input Mapping):**

  - Map mouse pixel $(x_p, y_p)$ to world $(x_w, y_w)$:

  $$
  x_w = \text{WorldMinX} + \frac{x_p}{\text{ScreenWidth}} (\text{WorldMaxX} - \text{WorldMinX})
  $$

  $$
  y_w = \text{WorldMaxY} - \frac{y_p}{\text{ScreenHeight}} (\text{WorldMaxY} - \text{WorldMinY})
  $$

  Assuming pixel origin top-left and world origin bottom-left

- **DrawView:** Configure the Graphics API (e.g., `glOrtho` in OpenGL) using `WorldBounds` instead of fixed screen dimensions.
