### Answer 21.10
    
**Virtual Sphere Design:**

1.  **Conceptual:**
    * **Metaphor:** The object is encased in a glass sphere that can be rotated by friction.
    * **Objects:** Scene object, Virtual Sphere (proxy).
    * **Operations:** Rotate sphere (and thus object).

2.  **Functional:**
    * **Input:** 2D locator device (mouse).
    * **Output:** Updated 3D orientation matrix.
    * **Feedback:** Visual proxy (sphere wireframe) appears on hover/click.

3.  **Sequencing:**
    * State: *Idle*. Event: `RightMouseDown` on object $\to$ State: *Active* (Show Sphere).
    * State: *Active*. Event: `LeftMouseDown` on sphere $\to$ State: *Dragging* (Record $P_{start}$).
    * State: *Dragging*. Event: `MouseMove` $\to$ Update Rotation ($R(P_{start}, P_{curr})$).
    * State: *Dragging*. Event: `LeftMouseUp` $\to$ State: *Active* (Apply $R$ permanently, reset $P_{start}$).
    * State: *Active*. Event: `RightMouseDown` off object $\to$ State: *Idle* (Hide Sphere).

4.  **Lexical:**
    * **Tokens:** Mouse coordinates $(x,y)$, Button IDs.
    * **Mapping:** Map screen $(x,y)$ to sphere surface $(x,y,z)$ via ray casting.
    * **Feedback:** Cursor change (hand icon), sphere highlight.
