### Answer 2.5

**(a) Observation:**

Shrinking the window clips the image because `ClipToBounds="True"` prevents drawing outside the allocated canvas area. The origin $(0,0)$ remains fixed at the top-left, so the bottom-right of the clock disappears first.

**(b) Adaptation Strategy:**

- **Zoom-out:** Apply a dynamic `ScaleTransform` to the Canvas based on the window size. If the window shrinks by 50%, reduce the ScaleX/ScaleY by 50%.
- **Pan/Scroll:** Use `TranslateTransform`. Capture mouse drag events to update the `X` and `Y` values of the transform, effectively moving the scene viewport relative to the window.
