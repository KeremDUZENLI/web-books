### Answer 14.1
**Floating Point Non-Associativity:**
In floating-point arithmetic, precision limits can cause $(a + b) + c \neq a + (b + c)$.
Example: Let $a = 1.0 \times 10^{30}$, $b = -1.0 \times 10^{30}$, and $c = 1.0$.
* $(a + b) + c = (10^{30} - 10^{30}) + 1 = 0 + 1 = 1$.
* $a + (b + c) = 10^{30} + (-10^{30} + 1)$.
    If the floating-point mantissa does not have enough precision to represent $10^{30}$ and $1$ simultaneously (requires ~100 bits, standard float has 23), $(-10^{30} + 1)$ will round to $-10^{30}$.
    Result: $10^{30} - 10^{30} = 0$.
Thus, $1 \neq 0$.
