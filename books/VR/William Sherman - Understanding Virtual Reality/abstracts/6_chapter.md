# Chapter 6: Presenting the Virtual World

---

## Main Idea

- This chapter divides the creation of sensory stimuli into two stages: **Representation** (how the world is modeled) and **Rendering** (how that model is generated by hardware/software).
- It emphasizes the critical constraint of **real-time performance**, noting that maintaining an adequate frame rate often requires tradeoffs between visual complexity (fidelity) and rendering speed to preserve the illusion of reality.

---

## Key Principles & Guidelines

**Representation vs. Rendering**

- **Definition:** Representation is the data structure used to describe the virtual world (e.g., polygon meshes, voxels, mathematical surfaces). Rendering is the process of converting that data into sensory output (pixels, sound waves, force) in real time.
- **Design Implication:** Developers must choose a representation format that the rendering engine can process efficiently. High-detail models (representation) must often be optimized (LODs, culling) during the rendering phase to prevent lag.

**Visual Rendering Pipeline**

- **Definition:** The sequence of steps to draw a 3D scene:
    1.  **Application Stage:** Logic, physics, and input handling.
    2.  **Geometry Stage:** Coordinate transformations, lighting, and clipping (removing objects outside the view).
    3.  **Rasterization Stage:** Converting 3D geometry into 2D pixels on the screen.
- **Design Implication:** Bottlenecks can occur at any stage. If the CPU is overloaded with physics (Application), the GPU waits; if the scene has too many polygons (Geometry), the frame rate drops. Balance is key.

**Level of Detail (LOD)**

- **Definition:** A technique where an object has multiple representations with varying polygon counts. High-detail versions are used when the object is close; low-detail versions are used when it is far away.
- **Design Implication:** Essential for performance. It saves processing power by not drawing details the user cannot see, maintaining a stable frame rate without noticeable visual degradation.

**Culling Techniques**

- **Definition:** Methods to avoid calculating/drawing objects that aren't visible.
    - **Backface Culling:** Removing the "back" side of polygons facing away from the camera.
    - **View Frustum Culling:** Ignoring objects outside the camera's field of view.
    - **Occlusion Culling:** Not drawing objects hidden behind other objects.
- **Design Implication:** Efficient culling is the single most effective way to improve performance in complex environments.

**Collision Detection**

- **Definition:** The computational process of determining when two virtual objects intersect. It is critical for physics, interaction, and preventing users from walking through walls.
- **Design Implication:** Precise collision (mesh-to-mesh) is computationally expensive. Developers use "bounding volumes" (spheres, boxes) as a first pass to quickly rule out non-colliding objects before doing detailed checks.

---

## Practical Use

**Sonic Rendering (Auralization)**

- Just as visual rendering draws pixels, sonic rendering calculates sound. Simple panning (left/right) is insufficient for VR. True 3D audio requires calculating attenuation (volume drop over distance), Doppler shift (pitch change with motion), and spatialization (HRTFs) to place sound in 3D space.

**Haptic Rendering**

- Haptic loops must run much faster than visual loops (often 1000Hz vs. 90Hz) because the sense of touch is highly sensitive to temporal discontinuities. If the update rate is too slow, a solid wall will feel "mushy" or vibrating.

**Polygon Budgeting**

- A "budget" is set for how many polygons can be visible at once based on target hardware. Design the world to naturally occlude distant views (e.g., winding corridors instead of open plains) to keep the visible count within limits.
