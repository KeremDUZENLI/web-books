# Chapter 36: Getting Started

---

## Main Idea

- This concluding chapter serves as a call to action, urging readers to stop planning and start creating.
- It provides a practical, seven-week roadmap for a novice to transition into a VR creator, emphasizing that in a rapidly evolving field, the best way to learn is by doing, prototyping, and iterating immediately.

---

## Key Principles & Guidelines

**The "Just Do It" Philosophy**

- **Definition:** VR technology moves too fast for traditional, lengthy planning phases.
- **Design Implication:** A prototype should have been completed "last week." Lack of software experience is no excuse, as modern tools allow for point-and-click creation. The goal is to build, test, and iterate rather than perfect a design document.

**The 7-Week Roadmap**

- **Week 1: Experience VR**
    -   Buy a low-cost HMD (like Google Cardboard) if you don't have one.
    -   Attend a **VR Meetup** to try demos and meet other developers. Experience is the best teacher.
- **Week 2: Learn the Tools**
    -   Download a game engine (Unity or Unreal Engine).
    -   Complete basic "Hello World" tutorials to learn the interface.
- **Week 3: Learn the Design**
    -   Read this book (or other design resources) to understand the "Why" and "How" before getting too deep into bad habits.
- **Week 4: The First Build (Define-Make-Experiment)**
    -   Create a simple scene (a plane, a cube, a light).
    -   Build it and view it in the HMD.
    -   Modify it and view it again. This completes the first iterative loop.
- **Week 5: Define a Concept**
    -   Write a basic concept for a project.
    -   Ignore constraints and details; focus on creativity.
- **Week 6: Build a Prototype**
    -   Implement low-level functionality (grey-box phase).
    -   Show it to friends and family for immediate feedback.
- **Week 7+: Iterate**
    -   Expand on the stages or start over. The cycle of **Define -> Make -> Learn** continues indefinitely.

---

## Practical Use

**Community Engagement**

- The chapter highlights that even a simple demo built by a teenager can impress users. Connecting with the community (via Meetups) and sharing early work is critical for growth and motivation in the VR field.
