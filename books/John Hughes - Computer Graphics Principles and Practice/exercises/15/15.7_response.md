### Answer 15.7
    
**(a) Interval Arithmetic:**
Let $R \cap A$ be a set of intervals $I_A$ (entry $t_{in}$ and exit $t_{out}$ pairs) along the ray.
Let $R \cap B$ be a set of intervals $I_B$.

* **Union ($A \cup B$):** The ray intersects the union where it is inside A **OR** inside B. This corresponds to the merging of intervals $I_A \cup I_B$.
* **Intersection ($A \cap B$):** The ray intersects the intersection where it is inside A **AND** inside B. This corresponds to the overlap of intervals $I_A \cap I_B$.
* **Difference ($A - B$):** The ray intersects the difference where it is inside A **AND NOT** inside B. This corresponds to subtracting intervals $I_A - I_B$.

**(b) Ray-CSG-Tree Algorithm:**
```cpp
struct Interval { float t_in, t_out; };
typedef std::vector<Interval> IntervalList;

IntervalList TraceCSG(Ray R, Node node) {
    // 1. Transformations: Transform ray into node's local space
    if (node.transform) R = node.inverseTransform * R;

    // 2. Leaf Case: Primitives
    if (node.isPrimitive) {
        return IntersectPrimitive(R, node.shape); // Returns sorted intervals
    }

    // 3. Recursive Step
    IntervalList Left = TraceCSG(R, node.childA);
    IntervalList Right = TraceCSG(R, node.childB);

    // 4. Combine Intervals
    if (node.op == UNION) return MergeIntervals(Left, Right);
    if (node.op == INTERSECTION) return IntersectIntervals(Left, Right);
    if (node.op == DIFFERENCE) return SubtractIntervals(Left, Right);
}

```

The final image pixel is determined by the smallest  in the returned interval list.
