### Answer 15.6
    
**Instancing via Coordinate Systems:**
Instead of transforming the geometry (1000 copies), we transform the rays into the local coordinate system of the single static teapot model.

**Representation:**
Store the scene as:
1.  One canonical `Mesh` (the teapot) in local space.
2.  A list of `Instance` objects. Each `Instance` contains:
    * A reference to the `Mesh`.
    * A transformation matrix $M$ (Model-to-World).
    * The inverse matrix $M^{-1}$ (World-to-Model).

**Tracing Algorithm:**
For each ray $R_{world}$ with origin $O$ and direction $D$:
1.  Iterate over all `Instances`.
2.  Transform the ray to local space: 
    $R_{local}.origin = M^{-1} \times O$
    $R_{local}.direction = M^{-1} \times D$
3.  Intersect $R_{local}$ with the canonical `Mesh`.
4.  If an intersection occurs at time $t$:
    * The intersection point $P_{world} = R_{world}(t)$.
    * The normal must be transformed using the inverse transpose: $N_{world} = (M^{-1})^T \times N_{local}$, then normalized.
5.  Select the intersection with the smallest positive $t$ across all instances.
