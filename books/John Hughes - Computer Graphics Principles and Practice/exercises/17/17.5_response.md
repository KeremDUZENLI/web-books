### Answer 17.5

**Problem:** Random access is expensive; row access is cheap. Memory is limited.
**Goal:** Generate downsampled rows for MIP levels $k+1$ using only rows from level $k$ efficiently.

**Efficient Row-Based Algorithm:**
1.  **Buffer:** Maintain a small buffer in memory to hold just **two rows** of the current source MIP level $k$.
2.  **Process:**
    - Read Row $2i$ and Row $2i+1$ of level $k$.
    - **Compute:**
        - Generate one output row for level $k+1$ by averaging pixels vertically (between the two buffered rows) and horizontally (adjacent pixels).
    - **Write:** Immediately write this generated row to the output file/stream for level $k+1$.
    - **Discard:** Discard the two source rows from memory.
3.  **Repeat:** Continue until all rows of level $k$ are processed to produce level $k+1$. Then repeat the entire process using the stored level $k+1$ as the new source for level $k+2$.

**Efficiency:**
- **Memory:** $O(\text{width})$ (stores only 2 rows).
- **I/O:** Sequential read/write (minimizes seek costs).
- **Computation:** Each pixel is read exactly once per level generation.
