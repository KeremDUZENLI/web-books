### Answer 21.9
    
**Implementation:**
* **Integral:** $R_{total} = Rotation(P_{start} \to P_{curr})$.
* **Incremental:** $R_{total} = Rotation(P_{prev} \to P_{curr}) \circ R_{total}$.

**Experiment (Closed Loop):**
Move mouse $P_{start} \to \dots \to P_{start}$.
* **Integral:** Returns exactly to initial orientation (state depends only on current mouse pos).
* **Incremental:** Likely **does not** return to initial orientation. Rotations in 3D do not commute. The sequence of small rotations $R_1 R_2 \dots R_n$ is path-dependent. A loop on the sphere creates a net rotation (holonomy), leaving the object spun around the view axis.

**Preference:**
* **Integral** is generally preferred for object inspection because it is predictable and stable (resetting the mouse resets the object).
* **Incremental** feels more like physically pushing a ball with friction, which can be useful for spinning an object freely, but harder to control precisely.
