### Answer 27.1
    
**(a) Barycentric equivalence:**
Interpolating values linearly along edges and then linearly across scanlines describes a bilinear interpolation relative to the screen axes. A triangle defines a unique plane in $(x, y, \text{value})$ space. Both barycentric interpolation and the scanline algorithm calculate the $z$-coordinate (value) of this plane for a given $(x, y)$. Since three points uniquely define a plane, and both methods produce planar values that match at the vertices, they must yield the same value for any internal point $(x, y)$, provided the edge intersections are computed exactly.

**(b) Constant difference per row:**
Let the value being interpolated be $C$. The rate of change of $C$ along an edge is constant ($\frac{dC}{dy}$). If pixel rows are spaced by a constant $\Delta y = 1$, the change in the starting value $C_{start}$ from row $i$ to row $i+1$ is $\Delta C = \frac{dC}{dy} \Delta y$, which is constant for a straight edge.

**(c) Implementation:**
This suggests a Digital Differential Analyzer (DDA) approach.
1.  Sort vertices by $y$.
2.  Calculate slopes for the edges: $m_1 = \frac{x_2 - x_1}{y_2 - y_1}$, $dm_{c1} = \frac{c_2 - c_1}{y_2 - y_1}$, etc.
3.  Initialize `current_x` and `current_c` at the top vertex.
4.  For each scanline:
    * Plot pixel at `current_x`.
    * Increment `current_x += m`.
    * Increment `current_c += dm_c`.
    * Fill the span between left and right edges using a similar horizontal incremental approach.

**(d) Rotation of Quadrilaterals:**
**No.** Bilinear interpolation over a quadrilateral depends on the orientation of the scanlines relative to the quad's vertices. If the quad is not a parallelogram (i.e., not an affine transformation of a square), the interpolation is not rotationally invariant. Rotating the quad changes which vertex pairs define the "left" and "right" edges and the scanline directions, resulting in a different interpolated surface (hyperbolic paraboloid) that warps differently.
