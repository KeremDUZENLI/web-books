### Answer 8.5

**Portal System Implementation:**

1.  **Data Structure:**
    * Define `Zone` containing geometry and a list of `Portal` objects.
    * Define `Portal` containing a convex polygon vertices and a link to the connected `Zone`.

2.  **Recursive Visibility Function `Traverse(Zone, Frustum)`:**
    * Mark `Zone` as visible.
    * Render geometry in `Zone` against `Frustum`.
    * For each `Portal` in `Zone`:
        * **Cull**: Check if `Portal` polygon intersects `Frustum`. If not, continue.
        * **Clip**: Clip `Portal` polygon against `Frustum` planes (Near, Far, Left, Right, Top, Bottom) to generate a reduced polygon.
        * **Reduce Frustum**: Construct a new `Frustum'` using the camera position and the edges of the clipped portal polygon (Equation 8.56).
        * **Recurse**: Call `Traverse(ConnectedZone, Frustum')`.

3.  **Clipping Logic:**
    * Sutherland-Hodgman algorithm: Clip polygon against each plane sequentially.
    * For each edge vertex pair $(V_i, V_{i+1})$:
        * If both inside, output $V_{i+1}$.
        * If out $\to$ in, output intersection point and $V_{i+1}$.
        * If in $\to$ out, output intersection point.
        * If both outside, output nothing.
